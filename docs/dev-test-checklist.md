# Dev Branch Test Checklist

适用分支：`origin/dev`

目标：在不直接改动线上插件目录的前提下，验证这一轮工作是否在真实 AstrBot 运行时符合预期。

## 部署前确认

- AstrBot 版本不低于 `4.5.7`
- 插件代码来自 `https://github.com/Alterego-42/astrbot-luogu-fetcher/tree/dev`
- 本轮测试不要直接修改当前线上插件目录，由你自己决定如何拉取和装载 `dev`

## 测试一：`/luogu jump` 基础流程

- 执行 `/luogu jump`
- 选择一个难度
- 添加 1 到 2 个标签
- `done` 后可选填关键词，也可以 `skip`
- 确认能进入候选结果页
- 测试 `random`
- 测试 `back`
- 预期：
  - 整个流程可正常走完
  - `back` 会回到合理的上一步，而不是把会话打乱

## 测试二：当前题目的 `看图` / `截图`

- 在 `/luogu jump` 里选出一题
- 发送 `看图`
- 再发送 `截图`
- 然后继续发送 `random` 或 `back`
- 预期：
  - `看图` 发送的是 Markdown 渲染长图，而不是整页网页截图
  - LaTeX 公式能正常显示
  - `截图` 主要截到题面主体，而不是整页乱截或空白
  - 发完图片后仍保留当前会话上下文，不会掉回错误状态

## 测试三：`/luogu jump` 内自然语言选题

- 在 jump 会话里直接说：
  - `来一道 ICPC 图论题`
  - `再来一道次小生成树的题`
  - `第 3 题`
- 预期：
  - 自然语言条件能转成筛选条件
  - “次小生成树”不会被粗暴扩成泛化的“生成树”标签筛选
  - 如果标签无法精确命中，会提示或回退为关键词，而不是静默放大范围
  - `第 3 题` 在已有候选上下文中能正常工作

## 测试四：普通聊天中的全局选题工具

- 在普通聊天里直接提出选题请求，例如：
  - `来一道 ICPC 图论题`
  - `找几道 O2 优化题`
  - `随便来一道`
- 再尝试这些边界表达：
  - `截图`
  - `第 3 题`
- 预期：
  - 主 LLM 能在合适时机调用 `luogu_problem_search`
  - 返回候选题目或具体题目时，内容应包含题号、标题、链接，必要时还有难度/标签
  - 单独说 `截图` 时，不应误触发整站搜索，而应提示改用 `/luogu jump`
  - 单独说 `第 3 题` 时，不应假装记住上一轮候选列表，而应提示补充条件或改用 `/luogu jump`

## 回传建议

如果发现问题，优先回传以下任一形式：

- 你的推测总结
- 聊天记录
- 运行日志

如果能同时提供“触发语句 + 实际回复 + 日志片段”，后续定位会更快。
